# 🚀 DriveMind Edge AI Car OS — Hackathon MVP Guide

Welcome to the **DriveMind Edge AI Car OS** repository! We have restructured and expanded your codebase into a next-generation local AI vehicle operating system.

Instead of a basic "LLM chatbot inside a car," DriveMind acts as a proactive **AI Co-Pilot** that:
1. Runs **100% offline** (Edge AI) using a local Ollama instance (`phi3:mini`) and embeddings (`nomic-embed-text`).
2. Maintains a **persistent local profile** memory for family members (Dad, Mom, Child) to deliver true personalization.
3. Automatically executes **vehicle simulator commands** (adjusting AC, opening/closing windows, toggling headlights/hazard lights, starting route navigation) using structured parsing.
4. Explains dashboard warning lights (TPMS, Tortoise Mode, etc.) by retrieving context from the vehicle's manual (RAG) using a persistent local ChromaDB instance.
5. Employs a **polished, dark-themed automotive dashboard** with visual indicators, gauge meters, and manual simulator override buttons to trigger predictive AI actions on demand.


## ⚡ Core Features Implemented

1. **Natural Co-Driver Personality**:
   - Persona: "DriveMind", an alert, empathetic assistant.
   - Grounded responses with precise explanation (e.g., "I've routed you through Eastern Freeway to save 15 minutes").
2. **Proactive & Predictive Alerts**:
   - **Low Fuel/Battery**: If battery ≤ 15%, alerts driver. If ≤ 10%, displays a "Tortoise Mode" alert and caps speed at 40 km/h.
   - **Rain Sensor**: Proactively suggests headlights/wipers.
   - **Fatigue Sensor**: If driving for > 2 hours, detects sleepiness, alerts driver, and offers GPS navigation to the nearest coffee shop.
3. **Owner Manual RAG**:
   - Automatically parses `tata_nexon.pdf` (or `car_manual.txt` fallback) into a persistent `data/chroma_db` database on first run.
   - Instant answers on complex queries: "Why is my TPMS light flashing?", "Where is the fuse box?", "What is Tortoise Mode?".
4. **Function Calling**:
   - Commands like `set_ac_temp(temp=21.0)`, `set_windows(open_win=true)`, `play_music(playlist='evening')`, and `start_navigation(destination='Home')` are generated by the LLM and executed by the simulator.
5. **Emergency Mode**:
   - Responding to "I had an accident", the AI enters emergency state: activates hazards, unlocks doors, calls the profile's emergency contact, and shares coordinates & medical information.
6. **Family Memory Profiles**:
   - Dad, Mom, and Child profile data stored in `data/user_profile.json`. Switching profile immediately updates seating position, favorite playlist, and climate temperature.

## 🚦 How to Run the Project (Step-by-Step)

### Prerequisites
1. **Ollama**: Verify Ollama is running and has both required models pulled:
   ```powershell
   ollama pull phi3:mini
   ollama pull nomic-embed-text
   ```
2. **Python Environment**: Ensure the virtual environment (`venv`) is activated.
3. **Flutter Toolchain**: Flutter is installed and functional.

### Step 1: Boot the Local API Backend
Run the root startup script from the workspace directory:
```powershell
.\venv\Scripts\python main.py
```
*Note: On the first run, the system will detect `tata_nexon.pdf` and extract/vectorize pages to save to `data/chroma_db`. Subsequent startups will load the database instantly (under 1 second).*

### Step 2: Run the Flutter Dashboard App
Open a separate terminal window and start the Flutter application. You can run it on Chrome or Edge (web browser) or compile it natively for Windows:

**To run in Chrome web browser (Highly Recommended for offline demo):**
```powershell
cd frontend
flutter run -d chrome
```

**To run as a Windows Desktop app:**
```powershell
cd frontend
flutter run -d windows
```

## 🎭 The Winning Judge Demo Script

To prove true **Edge AI** capability, follow this script during your presentation:

1. **Setup**: Boot the FastAPI backend and open the Flutter dashboard.
2. **True Offline Proof**: **Turn off Wi-Fi on your laptop.**
3. **Step 1: Welcome & Hello**
   - Click the chip: **"Hello"**
   - *AI response: Introduces itself as DriveMind, running offline inside the car.*
4. **Step 2: Proactive Climate Control**
   - Click the chip: **"I'm cold"** or type it.
   - *AI response: Dynamically increases AC temperature. You will see the climate control dial update immediately on the dashboard UI!*
5. **Step 3: Owner Manual RAG**
   - Click the chip: **"Why is my TPMS light on?"**
   - *AI response: Connects to ChromaDB, pulls SECTION 1 of the manual, and explains that front tires should be 34 PSI, rear tires 32 PSI. Flashing for 1 minute then solid indicates a system malfunction.*
6. **Step 4: Predictive Trigger (Sleepy Driver)**
   - Click the **"Sleepy Driver"** simulation button at the bottom of the dashboard.
   - *AI response: Detects high driving fatigue, alerts the driver, and asks to navigate to a coffee shop 4 km away.*
7. **Step 5: Navigation & Music**
   - Click the chip: **"Play my evening playlist"** and then **"Navigate home"**
   - *AI response: Music starts playing (vinyl record begins spinning!) and route display updates to Home.*
8. **Step 6: Personal Memory Display**
   - Type: **"What have you learned about me?"**
   - *AI response: Details the active profile's preferences (e.g. Dad prefers Sport mode, 21°C AC, A.R. Rahman Hits, and avoids tolls).*
9. **Step 7: Profile Switch**
   - Click **"Profile: Mom"** on the testing console.
   - *Result: The dashboard seating shifts to memory 2, AC adjusts to 23°C, and drive mode switches to Comfort.*

## 🏆 Hackathon Winning Tips
- **Keep Internet Off**: The judges will see the dashboard running and responding instantly while the laptop is completely disconnected from Wi-Fi. This creates massive impact.
- **Explainability**: Point out how the navigation avoids tolls because "Dad's profile has `avoid_tolls: true` set."
- **Flawless Safety Fallback**: While the app supports physical microphone inputs, the **testing console scenario buttons** and **quick chips** ensure that your demo is 100% immune to noise-related speech errors during a loud hackathon hall!
